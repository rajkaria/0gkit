# K6 — `0g mcp init <agent>`

## Goal

Make 0gkit **native to your AI tool** in one command. After K6:

```bash
0g mcp init cursor        # writes .cursor/mcp.json with the 0gkit-mcp server
0g mcp init claude        # writes ./.mcp.json (project) — Claude Code + desktop variants
0g mcp init windsurf      # writes .windsurf/mcp.json
0g mcp init codex         # writes .codex/mcp.json
0g mcp init claude --global   # writes the agent's user-level config instead
```

The generated config exposes the full neutral 0gkit tool set (the eight tools
already shipped in `0gkit-mcp`). **K0 synergy:** kits that ship MCP adapters
(`agent-memory`, `durable-agent`, `sealed-inference`) auto-register their tools
into the generated config.

## Dependencies / Architecture

- **`0gkit-mcp` already exists** (SP12-era) and exports the neutral `TOOLS` array
  (`og_storage_put`, `og_storage_get`, `og_storage_exists`, `og_infer`,
  `og_da_publish`, `og_da_verify`, `og_chain_faucet`, `og_chain_balance`,
  `og_attest_verify`) and a stdio `server`
  ([packages/0gkit-mcp/src/tools.ts](../../../packages/0gkit-mcp/src/tools.ts),
  [packages/0gkit-mcp/src/cli.ts](../../../packages/0gkit-mcp/src/cli.ts)). K6
  does **not** add tools to the server — it generates the _editor config_ that
  points each agent at that server.
- **The config writer is a pure function** in a new `0gkit-mcp` module
  (`config-init.ts`): `(agent, scope, kitTools) → { path, json }`. The CLI command
  `0g mcp init` is a thin shell that lazy-imports it (D39 — keeps cold-start fast,
  same pattern `0g jobs`/`0g test` use), resolves paths, and writes via the
  injected `ProgramDeps.fs`.
- **Per-agent config shapes** (each writes the standard `mcpServers` map keyed to
  the `npx @foundryprotocol/0gkit-mcp` stdio command):
  - `cursor` → `.cursor/mcp.json` (global: `~/.cursor/mcp.json`)
  - `claude` → project `./.mcp.json` (global: `~/.claude/mcp.json`)
  - `windsurf` → `.windsurf/mcp.json` (global: `~/.codeium/windsurf/mcp_config.json`)
  - `codex` → `.codex/mcp.json` (global: `~/.codex/mcp.json`)
- **Kit MCP-tool registration (K0 synergy):** read `.0gkit/kits.json` (the K0
  `ApplyResult` manifest). For each applied kit whose `kit.json` declares an
  `mcp-agent` adapter, append its declared tool ids to the generated config's
  server `env` (`OGKIT_MCP_KITS=agent-memory,durable-agent`) so the running
  server enables those tools. Engine-neutrality is preserved: `0gkit-mcp` reads
  the kit _manifest names_, it does not import the kit overlays.
- New docs page `apps/docs/app/concepts/mcp/page.mdx` (joins the existing
  `concepts/durable-jobs` + `concepts/observability` pages).

```ts
import { ConfigError } from "@foundryprotocol/0gkit-core";

export const AGENTS = ["cursor", "claude", "windsurf", "codex"] as const;
export type AgentName = (typeof AGENTS)[number];
export type McpScope = "project" | "global";

interface AgentPaths {
  project: string;
  global: string;
}
const PATHS: Record<AgentName, AgentPaths> = {
  cursor: { project: ".cursor/mcp.json", global: ".cursor/mcp.json" },
  claude: { project: ".mcp.json", global: ".claude/mcp.json" },
  windsurf: {
    project: ".windsurf/mcp.json",
    global: ".codeium/windsurf/mcp_config.json",
  },
  codex: { project: ".codex/mcp.json", global: ".codex/mcp.json" },
};

export interface McpServerEntry {
  command: string;
  args: string[];
  env?: Record<string, string>;
}

export function buildMcpConfig(opts: {
  agent: AgentName;
  scope: McpScope;
  home: string;
  cwd: string;
  kitTools?: string[];
}): { path: string; json: { mcpServers: Record<string, McpServerEntry> } } {
  if (!AGENTS.includes(opts.agent))
    throw new ConfigError(
      `Unknown agent '${opts.agent}'.`,
      `Use one of: ${AGENTS.join(", ")}.`
    );
  const rel = PATHS[opts.agent][opts.scope];
  const base = opts.scope === "global" ? opts.home : opts.cwd;
  const env: Record<string, string> = opts.kitTools?.length
    ? { OGKIT_MCP_KITS: opts.kitTools.join(",") }
    : {};
  return {
    path: `${base}/${rel}`,
    json: {
      mcpServers: {
        "0gkit": {
          command: "npx",
          args: ["-y", "@foundryprotocol/0gkit-mcp"],
          ...(Object.keys(env).length ? { env } : {}),
        },
      },
    },
  };
}
```

Export `buildMcpConfig`, `AGENTS`, `AgentName`, `McpScope` from `src/index.ts`.

```ts
import type { Command } from "commander";
import { runCommand, type ProgramDeps } from "../program.js";

export function registerMcp(program: Command, deps: ProgramDeps): void {
  const mcp = program.command("mcp").description("wire 0gkit into your AI editor");
  mcp
    .command("init <agent>")
    .description("write MCP config for cursor | claude | windsurf | codex")
    .option("--global", "install to the agent's user-level config (default: project)")
    .action(async function (this: Command, agent: string) {
      const opts = this.opts() as { global?: boolean };
      await runCommand(deps, this, async () => {
        // D39 — lazy specifier keeps `0gkit-mcp` out of CLI cold-start.
        const mod = await import(/* @vite-ignore */ "@foundryprotocol/0gkit-mcp");
        const kitTools = await mod.readKitMcpTools(deps.cwd(), deps.fs);
        const { path, json } = mod.buildMcpConfig({
          agent,
          scope: opts.global ? "global" : "project",
          home: deps.homedir(),
          cwd: deps.cwd(),
          kitTools,
        });
        await deps.fs.mkdir(path.slice(0, path.lastIndexOf("/")));
        await deps.fs.writeFile(path, JSON.stringify(json, null, 2) + "\n");
        return {
          human: [
            `✓ wrote ${path}`,
            `  server: npx @foundryprotocol/0gkit-mcp`,
            ...(kitTools.length ? [`  kit tools: ${kitTools.join(", ")}`] : []),
            `  Restart ${agent} to pick up the 0gkit tools.`,
          ],
          json: { agent, path, kitTools },
        };
      });
    });
}
```

Add `homedir()` to `ProgramDeps` (default `os.homedir`); register `registerMcp(program, deps)` in `program.ts` after `registerTraces`.
